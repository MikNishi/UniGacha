ДЛЯ РАБОТЫ ИСПОЛЬЗУЕМ ВЕТКУ develop

1. Описание папок и для чего они нужны. Пожалуйста, старайтесь сохранять структуру папок. 
Если нужны дополнительные папки (чтобы разделить оружки/персов/артефакты) - создавайте, но в нужной директории.
Если какие-то папки считаете излишними - не используйте (смотри п. 4)

Assets/
├── Core                   # Общие вещи: сцены, менеджеры, настройки (CORE_)
│   ├── Scenes             # Основные сцены проекта
│   ├── Scripts            # Общие скрипты: GameManager, AudioManager и т.п.
│   └── Prefabs            # Общие префабы (UI, камеры и т.д.)
│
├── 3D   	                # Основной 3D-геймплей (3D_)
│   ├── Models             # 3D-модели
│   ├── Materials          # Материалы для 3D-сцены
│   ├── Animations         # Анимации для 3D-объектов
│   ├── Prefabs            # Префабы NPC, предметов, зданий
│   └── Scripts            # Скрипты логики для 3D
│
├── VisualNovel            # Сюжет и визуальная часть (VN_)
│   ├── Characters         # Спрайты/арты персонажей
│   ├── Backgrounds        # Фоны сцен
│   ├── Dialogues          # JSON, Ink или другие форматы диалогов
│   ├── UI                 # Интерфейс новеллы (окна, кнопки, текст)
│   └── Scripts            # Сценарии диалогов, системы переходов и т.п.
│
├── Roguelike              # Мини-игра	(RL_)
│   ├── Sprites            # 2D-графика
│   ├── Animations         # Анимации 2D-персонажей
│   ├── Scripts            # Логика рогалика
│   └── Prefabs            # Префабы 2D-объектов
│
├── Audio                   # Звуки и музыка (AU_)
│   ├── SFX                # Эффекты
│   └── Music              # Фоновые треки
│
├── Fonts                   # Шрифты
├── Resources               # Загружаемые во время игры ресурсы (если надо)
└── Plugins                 # Сторонние библиотеки

2. При создании файлов старайтесь добавлять префикс в начале названия. 
Например, префаб персонажа в рогалике: RL_char1.prefab, Core_Menu_Settings.prefab - префаб меню, RL_Ability_DoubleJump.cs - скрипт способности.
Чтобы было проще что-то искать и соотносить.

3. Не забывайте .meta файлы
4. Старайтесь не дублировать ассеты - если что-то общее, лучше поместить в Core.
5. Перед коммитом, на всякий случай, проверяйте, не закоммитили ли Library/Logs и тд.
